# React 19 + FSD (адаптированная архитектура)

## Структура проекта (адаптированная под FSD)

В проекте Neprikosnovenna используется модифицированная версия FSD. Основные слои:

- **`src/app/`** — настройки приложения (роутер, провайдеры, глобальные стили)
  - `App.jsx`, `AppRouter.jsx`, `AppRouter.config.js`
- **`src/pages/`** — страницы (слайсы)
  - `/neprikosnovenna` — статичный портрет
  - `/neprikosnovenna/and-i-am-the-only-one...` — видео-трансформация
- **`src/components/`** — переиспользуемые UI-компоненты (слайсы shared)
  - Каждый компонент в своей папке с `index.jsx`, `styles.module.scss`, `*Settings.js`
- **`src/hooks/`** — кастомные хуки (shared/lib)
- **`src/utils/`** — утилиты (shared/lib)

## Императивный паттерн (API компонентов)

Компоненты, предоставляющие императивное API, следуют шаблону:

```jsx
const Component = forwardRef((props, ref) => {
  const [state, setState] = useState()
  useImperativeHandle(ref, () => ({
    method1: () => { ... },
    method2: () => { ... }
  }))
  return <div>...</div>
})
```

Такие компоненты всегда должны:

- Иметь файл настроек `*Settings.js`, где определены типы и конфигурации
- Использовать `forwardRef` и `useImperativeHandle`
- Не смешивать императивный API с декларативной логикой
- **`displayName`** для удобства отладки:
  ```js
  Component.displayName = "ComponentName";
  ```
- **Типы методов** должны быть стабильными (не пересоздаваться при каждом рендере). Можно добавить замечание о том, что `useImperativeHandle` следует вызывать с постоянным объектом (использовать useMemo или стабильную ссылку).

## Курсорная система (физика + WebGL)

Физический курсор — отдельный слой логики:

- `useCursorMovePhysics` — spring-физика (stiffness, mass, damping)
- `useCursor` — оркестратор событий, анимационный цикл, детекция зон
- `useCursorZone` — маппинг элементов на зоны (NONE, BACK, PORTRAIT, BUTTON)

При работе с курсором:

- Не меняй напрямую CSS-курсор, используй систему зон
- Для новых интерактивных элементов добавляй атрибуты `data-cursor-zone` или классы, распознаваемые в `useCursorZone`

## Кодстайл

- **Отступы**: 4 пробела
- **Синтаксис**: точки с запятой не используются (consistent omission).
- **Именование**:
  - `camelCase` — переменные, функции, хуки
  - `PascalCase` — компоненты, файлы компонентов
  - `UPPER_CASE` — константы (реже)
  - `kebab-case` — директории
- **Обработчики событий**: префикс `handle` (`handleClick`, `handleSubmit`)
- **Булевы переменные**: префикс-глагол (`isLoading`, `hasError`, `canSubmit`)
- **Стили**: BEM + CSS Modules (`.module.scss`), desktop-first медиазапросы
- **React**: функциональные компоненты со стрелочными функциями, композиция вместо наследования, кастомные хуки для сложной логики
- **Сравнения**: строгие (`===`, `!==`)
- **Висячие запятые**: trailing commas

## Настройки компонентов

Каждый сложный компонент имеет файл `*Settings.js` (например, `cursorSettings.js`, `backgroundSettings.js`). Эти файлы содержат:

- Объекты конфигурации
- Фабрики для создания типов (cursorTypes, backgroundTypes, flashTypes, portraitTypes)
- Единый источник истины для вариантов компонента

При добавлении нового типа компонента — сначала расширь настройки, затем сам компонент.

## Роутинг (react-router-dom 7)

Страницы определяются в `AppRouter.config.js` как массив объектов с `path` и `component` (lazy-loaded). Не создавай страницы вручную — добавляй через конфиг.

## Производительность

Используй `React.memo` для компонентов, которые часто ререндерятся,`useMemo` / `useCallback` - для тяжёлых вычислений и функций, передаваемых в дочерние компоненты.

Ленивая загрузка страниц через `React.lazy`

Следи за размером бандла (Vite report).

Не создавайте анонимные функции в JSX (в пропсах) — это ломает мемоизацию.

Для функций, передаваемых в React.memo-компоненты, обязательно оборачивайте в useCallback.

Для объектов/массивов в пропсах используйте useMemo.

Избегайте передачи новых объектов/функций в контекст.
